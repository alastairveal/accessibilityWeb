Windows

The content in web pages can appear within browsers in a variety of different ways. This can be confusing, especially for screen reader users. This section below discusses the various ways that web pages can be opened and displayed.

Links

Links are connections to other web pages or parts of a web page. Clicking on the link or pressing the ENTER key when the link has the focus will activate the link. Links are normally coloured blue. Links can perform a range of different actions including opening a new page within the existing tab, opening a new page in a new tab, open an email client or open an overlay window. Tabs

A tab is a feature of web browsers that enables the user to have multiple web pages open within one browser window. This is to make navigating web pages easier, as users can quickly switch between multiple pages. To navigate between tabs using a mouse, users simply click on the tab at the top of the window. Alternatively press the CTRL & T to open a new tab Windows (Using the Firefox, Internet Explorer, Chrome and Edge browsers). Pressing and holding on the CTRL key and then pressing the TAB key will cycle to the next tab. On these browsers ALT and F4 will close the current tab. On a Mac computer using the Safari browser, COMMAND & T will open a new TAB, use COMMAND & SHIFT & LEFT or RIGHT ARROW to navigate between tabs and COMMAND & W to close the tab. Some links, instead of opening a new page within the existing window, will open a new page in another tab. Tabs should not be confused with the TAB key, which is used to cycle forward the focus to the next interactive element. Alerts

An alert window is a small window that pops up in front of the current window, to alert the user to something on the page, or to prompt the user to do something. Whilst the alert is open, the user cannot interact with the rest of the page. One of the buttons in the alert allows the user to close the alert (often this is a button titled ‘OK’). Alerts often used for form validation or when navigating away from a page where form fields have only been partially filled in. The content in an alert window is very basic. Modals

A modal, or an overlay window is a window that opens in front of the current window. When the modal or overlay window is open the content on the original window is covered with a grey mask. Unlike an alert, an modal or overlay window can support all the of the same kind of content as a standard web page. It is often used to provide extra information in a website to avoid making a website look too cluttered. Modalys or overlays can also be used to prompt the user to take certain actions, such as provide consent for cookies. There is often a close button on a modal or overlay, which is usually on the top right hand corner, and is often the first button on the overlay. Usually, but not always, when an overlay window is opened, the user cannot interact with the content behind the overlay. Depending on how the web page is designed, the web page may announce when a modal window is going to be opened. In a poorly designed page a modal window could open and a screen reader user is not aware that the modal has opened.

PDFs

PDFs are files which are primarily a mix of text and images. There are a number of accessibility issues with PDFs. Like a Word document or a book, PDFs are divided into separate printable pages. This means that the length of each page is limited, unlike web pages, which have no constraints on their size. PDFs are easy to create, the text in PDFs cannot easily be edited and they can be easily printed out. For these reasons it common for PDFs to be used on websites for reports, guides, tickets and journal articles. PDFs hosted on web pages will usually open in the browser window, often in new tab. When viewing a PDF in a browser, there a number of controls which the user can navigate to using the TAB key. The controls available will depend on the browser being used. Typical controls include:

  • Page Number: The user can type in the page number they want to view.
  • Page Rotation: The page can be rotated if the text is the wrong way up.
  • Download: To download the PDF and view offline.
  • Print: To print out the page.
  • Fit to Page: Zooms in the page as much as possible without losing content on the left or right hand sides.
  • Zoom: Zooms in and out.

Not all PDFs can be read aloud by screen reading software. Some PDFs are image only PDFs, meaning that the text in the PDF is an image of text, not actual electronic text. Image only files cannot be read aloud by screen reading software without using optical character recognition (OCR) to recognise the words in the PDF. If downloaded, the text of the image only PDFs can only be accessed by screen reading software users by using a document conversion application to convert the PDF either to a different format such as Word or converting the PDF to a PDF with electronic text. Newer versions of Microsoft Word will convert PDFs, including image only PDFs, to Word documents. Below is a link to an example of an inaccessible image only PDF.

In addition, PDFs can be challenging for partially sighted users who want to zoom in. It is possible to zoom in on PDFs in a browser (see section on zooming on the adjustments page). However PDFs have no word wrap. This means that as the user zooms in, content can be lost on the left and right hand sides. Content off the side of the visible area can be accessed using the LEFT or RIGHT ARROW keys, but this makes reading an awkward experience.

Downloading the PDF is useful because it can be viewed offline. Downloaded PDFs can be viewed using other applications such as Adobe Reader or Microsoft Word, which have more controls and options. If necessary, a downloaded PDF can also be converted to a different file format.

Frames with Text Content

Files with text based content, such as PDFs, can be embedded into a web page in a frame (section of a web page). This can be particularly challenging for accessibility reasons. Some frames do not have content that can be read aloud by screen reading software, because the screen reading software does not recognise the file format, or the file is image only text (a picture of text instead of electronic text). Screen reading software can access the text of PDF inside a frame, but it can be confusing for screen reader users, as the user is viewing one file format from within another file format. Often it is easier just to download text based content such as PDFs, which well designed pages will have as an option. If viewing a PDF in a frame, there are a number of controls such as page number or download. These are the same controls available when viewing a PDF in a browser (see section on PDFs above). To get to the actual content, screen reader users need to press the DOWN ARROW key until past the controls in the frame. Sighted users who cannot operate a mouse can also find text based content in frames difficult and it is often just easier to download the content if possible. This is because if the amount of content in the frame is longer than the length of the frame, the frame may have a scroll bar to navigate up and down the document. Getting the focus of the page to access the scroll bar of the frame can be difficult (The UP and DOWN ARROW keys can be used to move the content).
