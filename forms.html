Forms

A form on a web page allows the user to enter data. There are many kinds of form controls such as text boxes, comboboxes or buttons. Screen reading software will announce which kind of form control currently has the focus.

Text Input

A text input is a box that the user can type in data, such as name or address. Below is an example with first name and last name. Screen reading software will announce when the user is in an edit area and can type in text. Sometimes some edit areas automatically get the focus when the web page is opened (usually this is only used for search boxes).



Checkbox

A checkbox is a square box that can be ticked. It is used to select an option. When the checkbox has the focus, it can be checked and unchecked using the SPACEBAR key. The label for the checkbox normally appears after the checkbox (unlike most form controls where the label comes before the control).

Radio Button

A radio button is a small round control that can be selected. When the radio button has the focus, it can be selected and unselected using the SPACEBAR key, like the checkbox. Like the checkbox, the label for a radio button generally comes after the button. It is usually used to enforce one choice from a set of choices, like in the example below. Once the radio button group set has the focus, use the UP and DOWN ARROW keys to get to the next option. Pressing the TAB whilst a radio button has the focus will take the user to the next control, not the next option within the radio button set.

Drop Down Menu

A combobox, otherwise known as a drop down menu, provides the user with a list of choices. Once the drop down menu has the focus, the UP and DOWN ARROW keys can be used to select each choice. Alternatively, if the likely first letter of a choice is known, press on the letter of that choice. For example, in a list of countries, press the 'T' key to get to Taiwan. If there are several options of the same name, press the first letter of that option until the desired option is located. For example to get to the 'United Kingdom in a list of countries press the 'U' key four times, as the United Kingdom is the fourth choice after Uganda, Ukraine and the United Arab Emirates.

Button

A button is used to submit data or perform an action on the page. When the button as the focus, the SPACEBAR or ENTER key can be used to activate the button. Some buttons may be disabled, and will only become active after some kind of user activity (e.g. selecting a date).

Date Picker

Many forms require users to input dates. This typically consists of an input box, which the user can type in the date that they want. To help users get the right dates, date input boxes often have a date picker tool. This is a calendar that appears to help the user find the right dates. There are many different kinds of date picker calendars, so there is no one simple way in which a user who cannot use a mouse can access them. The simplest way to use a date input box for non-mouse users is just to type in the date, in the format required.

Below is an example of a date input box with a date picker. As the user types in a date a calendar pops up. It is not necessary to use the popup calendar, users can just type in the date needed using the DD/MM/YYYY format (e.g. 1st of January 2019 would be 01/01/2019). When the submit button is activated, the focus will return to the beginning of the page.

Range

A range, or slider, is a tool that lets the user choose from a number of different values. The range element is usually horizontal with a handle (bit that moves), and moving the handle to the left and right changes the value. For users who cannot use the mouse, using the LEFT and RIGHT ARROW keys lets the use the change the values when the range has the focus. Ranges can also be used as toggles between two options for example on and off. The slider below has 5 values, 1 to 5 and the handle starts at 3. Try to change the values.

Validation

Users make mistakes when inputting data. Forms have systems to check that the user has not put in any erroneous data. Typically when the user puts in incorrect data and then activates the submit button, instead of the data being submitted to the website, the form highlights to the user what information is wrong. Below is an example with a text box that is designed for the user to put in an email address. If the email address does not contain an ‘@’ symbol, when the submit button is activated, an error message will appear. Try entering an email address with and without and ‘@’ symbol. Note if the email address is valid the page will refresh and the focus will go to the top of the page.
